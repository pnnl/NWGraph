Class Documentation

template<bool is_const = false>
class soa_iterator

Member class defining the iterator for struct_of_arrays. We use a template parameter is_const to allow us to define both the iterator type and the const_iterator type.

Rather than maintaining a tuple of iterators (one for each underlying vector), we maintain a pointer to the struct_of_arrays and maintain a cursor in the iterator to indicate our current location. We use this approach for efficiency. Rather than incrementing a set of iterators when we advance the struct_or_arrays iterator, we only need to advance the cursor. This only works for the random access case, which we have for struct_of_arrays.
